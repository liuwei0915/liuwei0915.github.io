2022-02-28

Variable

variable "instance_type" {
  type =string
}
resource "aws_instance" "my_server" {
    ami = "ami-sadadafaf1030"
    instance_type = var.instance_type
}

then in the ‘terraform.tfvars’ file add:

instance_type = "t2.micro"

declaring a local value

locals {
  project_name = "Wei"    //or name can be anything
}

then for the tags session in the whole page could be write in:

tags = {
    Name = "allow_web.${local.project_name}"     //use local declared data
}

Output values

For example, output the public IP address.

//public_ip is just the name of output and can be any names you want
output "public_ip" {
  value = aws.instance.my_server.public_ip      //my_server is instance name
}

then we can use command “terraform output” or “terraform refresh” to see the output value

Modules

Go to website https://registry.terraform.io/browse/modules to find out the models
For example:

module "vpc" {
  source = "terraform-aws-modules/vpc/aws"
  provider = {
    aws = aws.us     //"us" is the alias of provider down below
  }
  name = "my-vpc"
  cidr = "10.0.0.0/16"

  azs             = ["us-east-1a", "us-east-1b", "us-east-1c"]
  private_subnets = ["10.0.1.0/24", "10.0.2.0/24", "10.0.3.0/24"]
  public_subnets  = ["10.0.101.0/24", "10.0.102.0/24", "10.0.103.0/24"]

  enable_nat_gateway = true
  enable_vpn_gateway = true

  tags = {
    Terraform = "true"
    Environment = "dev"
  }
}

then we can add provider for this vpc

provider "aws" {
  profile = "default"
  region = "us-east-1"
  alias  = "us"    
}

Terraform cloud workspace

Terraform.tfstate contains all of our informationabout the state our infrastructure and if we want to collaborate with other people we need to have it in shared space and so the terraform is one of the options and its the best option out there.
this is like the local backend of the terraform

  • terraform workspace list //will show the workspace which is default. we can define a remote backend that contains the terraform.tfstate informations in terraform cloud create the terraform cloud account with different workspaces.
terraform login //to find out the terraform api token file PATH for login
  • enter yes Then it will go to terraform to create token like cridential. Then copy and paste the value.
    then you can use:
terraform init

Initialize the terraform environment,

  • Do you want to copythe existing tate to the new backend. type yes
    it will move the terraform state file to your workspace
  • some variable is not able to function becuase the variable is local variable.
    so we need to manually add the variables into our terraform cloud workspace variables.
  • authentication environment vairables We also need to set up the environment vairbales if we need, such as: aws_access_key and secrete_key or default region.
  • Cleanup We can also delete the workspace in setting of terraform account

Terraform provisioners

  • Terraform provisioners install software, edit files. and provision machines created with Terraform.
    There are two different provisioners: Cloud-init and packaer
    1. cloud-init is an industry standard for cross-platform cloud instance initializations. Whne you launch a VM on a cloud service provider(CSP) you wil provide a YAML or Bash script.
    //userdata
#! /bin/bash
yum update -y 
yum install -y httpd
sudo service httpd start
sudo service httpd enable
    1. Packer is an automated image builder service. You privide a configuration file to create and provision the machine image and the image is the delivered to a repository for use.

cloud init script

users:
  - default
  - name: terraform
    gecos: terraform
    primary_group: hashicorp
    sudo: ALL=(ALL) NOPASSWD:ALL
    groups: users, admin
    ssh_import_id:
    lock_passwd: false
    ssh_authorized_keys:
      -  # Paste your created SSH key here
package_upgrade: yes
package_update: yes
packages:
  - httpd
runcmd: 
  - sudo service httpd start
  - sudo service httpd enable

Then we can define the template file

data "template_file" "user_data" {
  template = file("../scripts/add-ssh-web-app.yaml")
}

resource "aws_instance" "web" {
  ami                         = data.aws_ami.ubuntu.id
  instance_type               = "t2.micro"
  subnet_id                   = aws_subnet.subnet_public.id
  vpc_security_group_ids      = [aws_security_group.sg_22_80.id]
  associate_public_ip_address = true
  user_data                   = data.template_file.user_data.rendered

  tags = {
    Name = "Learn-CloudInit"
  }
}
